# 빙하의 이동 시뮬레이션

## 1. 문제 개요

$N \times N$ 크기의 바다에 존재하는 얼음 덩어리들이 매년 **융해, 이동, 병합**의 과정을 거칠 때, 특정 연도(1년 후)의 빙하 상태를 시뮬레이션하여 반환하는 문제.

* **빙하의 정의**: 상하좌우로 인접하여 연결된 얼음 덩어리들의 집합.
* **우선순위 조건**: 빙하 병합 시 이동 방향을 결정하는 우선순위는 `부피(최대) > 면적(최소) > Y좌표(최소) > X좌표(최소)` 순이다.
* **공간 조건**: 바다의 상하좌우 끝은 서로 연결되어 있는 토로이드(Toroidal) 구조이다.

## 2. 핵심 설계 전략

### 🚀 상태 분리 및 객체 지향적 우선순위 판별 (State Capture & Priority Queue)

융해 후 빙하가 갈라지거나 이동 후 합쳐질 때, 이동 방향 결정을 위해서는 **'이동하기 전' 빙하의 상태**가 필요하다. 

1. **상태 캡처**: 이동 직전에 BFS를 돌아 조각난 빙하들의 정보(ID, 부피, 면적, 기준 위치, 방향)를 `Piece` 객체로 생성해 둔다.
2. **Comparable 구현**: `Piece` 클래스 내부에 문제의 4가지 우선순위 규칙을 `compareTo`로 구현하여, 병합 시 `PriorityQueue`에 넣기만 하면 가장 강력한 빙하의 방향이 `peek()`되도록 설계한다.

### 🏠 2차원 배열 재사용 (Memory Optimization)

매년(최대 100회) 호출되는 `oneYearLater` 함수 내에서 배열이나 컬렉션을 매번 `new`로 생성하면 가비지 컬렉터(GC) 부하로 인해 시간 초과가 발생할 위험이 크다.

* **전역 배열 선언**: 최대 크기($100 \times 100$)의 2차원 배열들을 전역으로 한 번만 생성한다.
* **초기화 재사용**: 매 호출 시 `Arrays.fill()`과 `.clear()`를 통해 배열을 초기화하여 덮어씌우는 방식으로 메모리와 실행 시간을 극대화한다.

## 3. 알고리즘 프로세스

1. **초기화 (`init`)**:
* 입력받은 맵 정보를 배열에 복사.
* 주어진 빙하 위치에서 BFS를 수행하여, 연결된 하나의 빙하 덩어리 전체에 초기 이동 방향 배정.


2. **융해 (Melting)**:
* `map`을 순회하며 상하좌우 4방향 중 바다(높이 0)가 **하나라도 존재하면** 얼음 높이를 1 감소.
* 동시 다발적으로 일어나는 현상이므로, 원본 훼손을 막기 위해 `nextMap`에 계산 후 일괄 복사.


3. **이동 전 조각 식별 및 수집 (Capture Pieces)**:
* 융해 후 쪼개진 빙하들을 BFS로 탐색.
* 각 빙하 조각의 정보(부피, 면적 등)를 계산하여 `Piece` 리스트에 저장.


4. **이동 및 겹침 처리 (Move & Overlap)**:
* 모듈로 연산 `(좌표 + 방향 + N) % N`을 통해 공간의 끝이 연결된 이동 구현.
* 같은 좌표에 겹치는 경우 가장 높은 얼음(`Math.max`)만 남김.
* 해당 좌표에 도착한 조각의 ID를 `landed` 배열에 기록.


5. **최종 병합 및 방향 결정 (Merge & Decide)**:
* 이동 완료된 맵에서 다시 BFS를 돌아 서로 인접하게 된 거대 빙하 탐색.
* 탐색 중 마주치는 `landed`의 조각 ID들을 `PriorityQueue`에 삽입.
* PQ에서 최상단에 있는 조각의 방향을 꺼내어, 새롭게 병합된 빙하 전체의 방향으로 동기화.

## 4. 복잡도 분석

* **시간 복잡도**: $O(K \times N^2)$
* $N = 100$, 1년(`oneYearLater`) 시뮬레이션 시 전체 맵을 순회하는 상수 번의 BFS 수행 (약 $N^2$ 연산).
* 최대 호출 횟수 $K = 100$회 기준, 테스트 케이스당 약 $100 \times 10,000 = 10^6$번의 연산으로 제한 시간(2~3초) 내에 매우 넉넉하게 통과 가능.


* **공간 복잡도**: $O(N^2)$
* 크기 $100 \times 100$의 2차원 `int` 및 `boolean` 배열 다수 사용. 메모리 제한(256MB) 대비 극히 미미한 수준(수 MB 이내)으로 매우 안전.
