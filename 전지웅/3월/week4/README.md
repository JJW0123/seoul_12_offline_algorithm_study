# 대학 입학 배정 시뮬레이션

## 1. 문제 개요

$M$개의 대학에 지원하는 학생들의 성적과 지망을 바탕으로, 실시간 합격 및 대기 상태를 시뮬레이션하는 문제.

- **조건**: 각 대학 정원은 최대 $N$명이며 1지망부터 순차 선발.
- **우선순위**: `점수(최대) > 학생 ID(최소)` 순으로 합격자 결정.
- **핵심**: 학생 추가(`add`) 및 삭제(`erase`) 시 발생하는 연쇄적인 밀어내기와 끌어올리기를 제한 시간 내에 처리해야 함.

## 2. 핵심 설계 전략

### 🚀 이벤트 기반 부분 갱신 (Event-Driven Cascade)

매번 전체 학생을 재정렬(전수 조사)하지 않고, 변화가 발생한 지점의 연쇄 이동(도미노)만 처리하여 연산량을 극적으로 줄인다.

- **TreeSet (합격자 관리)**: 각 대학의 현재 합격자를 정렬 상태로 유지. 정원 초과 시 가장 스펙이 낮은 학생(`last()`)을 $O(\log N)$만에 밀어낸다.
- **PriorityQueue (대기열 관리)**: 거절당하거나 밀려난 학생만 큐에 보관. 빈자리가 생겼을 때 가장 우수한 대기자(`poll()`)를 $O(\log N)$만에 끌어올린다.

### 🏠 점수 캐싱 및 상태 관리 (State Optimization)

- **점수 캐싱**: `allScores` 2차원 배열에 각 학생의 대학별 환산 점수를 미리 계산해 두어 중복 연산을 방지.
- **소속 추적**: `res` 배열로 학생의 현재 소속을 $O(1)$로 즉시 파악 (1~$M$: 합격 대학, 0: 대기, -1: 삭제).

## 3. 알고리즘 프로세스

1. **학생 추가 및 밀어내기 (`add`)**:
   - 점수 계산 후 백수(`res = 0`) 상태로 1지망부터 문을 두드림.
   - **합격**: 정원 미달이거나 기존 꼴찌보다 우수하면 합격(`TreeSet.add`).
   - **도미노**: 밀려난 꼴찌는 대기열(`PriorityQueue`)에 들어가고 백수가 되어 다음 지망 대학으로 이동.

2. **학생 삭제 및 끌어올리기 (`erase`)**:
   - 학생을 삭제(`res = -1`)하고 소속 대학에서 지움.
   - 발생한 빈자리를 대기열의 최상위 적격자로 채움 (삭제된 학생, 상위 대학 합격자는 스킵).
   - 땡겨온 학생이 다니던 하위 대학에 다시 빈자리가 생기므로, 하위 대학을 기준으로 연쇄 끌어올리기를 반복.

3. **결과 조회 (`suggest`)**:
   - 시뮬레이션은 `add`와 `erase`에서 실시간 완료되므로, `res[mID]`를 $O(1)$로 즉시 반환.

## 4. 복잡도 분석

- **시간 복잡도**: $O(K \log (\text{학생 수}))$
  - 추가/삭제 시 발생하는 연쇄 이동을 트리와 힙 구조로 처리하여 기존 $O(N \log N)$의 재정렬 방식을 획기적으로 단축.
- **공간 복잡도**: $O(M \times \text{학생 수})$
  - 캐싱 배열(`allScores`)과 상태 배열(`res`), 노드 객체들만 사용하므로 제한 메모리 내에서 넉넉하게 동작.
